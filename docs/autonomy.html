Autonomy

Smart heartbeat-driven autonomy.

Context-aware scheduling with exponential backoff and action-oriented tasks.

Smart Heartbeat

The heartbeat system is now intelligent and context-aware:

  • Exponential Backoff: Intervals increase (2x, 4x, 8x) when unproductive to save resources
  • Productivity Tracking: Measures actual work vs idle cycles
  • Context Analysis: Examines recent conversations to determine relevant actions
  • Action Types: follow_up, outreach, research, maintenance, delegate

Backlog limit

Use autonomyBacklogLimit to cap queued proactive tasks.

Stall detection

Long-running actions are failed via maxActionRunMinutes.

Stale recovery

Stale in-progress actions are marked failed after maxStaleActionMinutes.

Multi-Agent Orchestration

For complex tasks, OrcBot can spawn worker processes:

  • Real Node.js child processes via fork()
  • IPC communication between main agent and workers
  • Shared configuration with isolated execution
  • View workers in TUI: orcbot ui → Workers menu
  • Skills: spawn_worker, list_workers, terminate_worker

Autonomy checklist

  • Enable autonomyEnabled
  • Set a heartbeat interval that fits your workload (auto-adjusts with backoff)
  • Monitor logs for proactive task output and worker activity
  • Use spawn_worker for parallel task execution
  • Check TUI Workers menu for process management
