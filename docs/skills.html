Skills & Plugins

Extend OrcBot with new tools.

Core skills live in the agent; plugins are hot-loaded without restarts.

Skill files (Markdown specs)

Skill specs are written in Markdown and can reference additional files via a Skill Files table. OrcBot will fetch all linked URLs when building the skill.

## Skill Files

    | File | URL |
    |------|-----|
    | **SKILL.md** | https://fredabila.github.io/orcbot/docs/skills/skill.md |
    | **HEARTBEAT.md** | https://fredabila.github.io/orcbot/docs/skills/heartbeat.md |
    | **MESSAGING.md** | https://fredabila.github.io/orcbot/docs/skills/messaging.md |
    | **package.json** | https://fredabila.github.io/orcbot/docs/skills/skill.json |

When the TUI skill builder sees this table, it downloads every linked file and uses the combined content to generate a plugin.

Core skills

Defined in src/core/Agent.ts. Key categories:

  • Web Search: web_search with smart API + browser fallback
  • Browser: browse_website, browser_action, browser_type
  • Messaging: send_telegram, send_whatsapp, reply_to_sender
  • Orchestration: spawn_worker, list_workers, terminate_worker
  • Learning: update_learning, update_user_profile, deep_reason
  • System: run_command, schedule_task, manage_skills

Search Resilience

The web_search skill uses smart provider ordering:

  • With API keys: Serper → Brave → SearXNG → DuckDuckGo → Bing → Google
  • Without API keys: DuckDuckGo → Bing → Google (browser-based only)
  • Multiple CSS selector strategies for each browser-based provider
  • Graceful fallback with actionable suggestions on total failure

Multi-Agent Skills

Orchestration skills for parallel task execution:

  • spawn_worker(name, task): Create a worker process for a specific task
  • list_workers(): View active worker processes with PIDs and status
  • terminate_worker(name): Stop a specific worker
  • broadcast_to_workers(message): Send instructions to all workers

Workers run as real Node.js child processes with IPC communication.

Skill Routing Rules

Configure intent-based skill selection via skillRoutingRules:

skillRoutingRules:
  - intent: "search"
    preferSkills: ["web_search", "browser_navigate"]
  - intent: "code"
    preferSkills: ["run_command", "manage_skills"]
  - intent: "messaging"
    preferSkills: ["send_telegram", "send_whatsapp"]

Routing rules help the agent select the most appropriate skills for each task type.
