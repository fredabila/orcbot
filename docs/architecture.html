Architecture

Understand the system layout.

Core orchestration, memory, tools, and external providers.

Core flow

Channels → Agent Core → Orchestrator → Workers → Skills & Tools → Providers
Telegram / WhatsApp / CLI • Multi-Agent • LLMs • Search (API + Browser) • Storage

Agent Core

DecisionEngine + SimulationEngine orchestrate every task with plans and guardrails.

AgentOrchestrator

Spawns worker processes via Node.js fork() for parallel task execution with IPC.

Smart Heartbeat

Context-aware scheduling with exponential backoff and productivity tracking.

Memory & Queue

MemoryManager stores short/episodic context. ActionQueue persists tasks.

Web Search

Resilient search with API providers (Serper, Brave) and browser fallback (DDG, Bing, Google).

Providers

MultiLLM routes to OpenAI, Gemini, or Bedrock with automatic fallback logic.

Data paths

  • ~/.orcbot: memory, profiles, plugins, downloads, worker configs
  • logs/: runtime logs (main agent + workers)
  • orcbot.config.yaml: configuration source

Key files

  • src/core/Agent.ts: Main agent with skills and heartbeat
  • src/core/AgentOrchestrator.ts: Multi-agent coordination
  • src/core/AgentWorker.ts: Worker process entry point
  • src/tools/WebBrowser.ts: Browser automation with resilient search
  • src/core/Scheduler.ts: Smart heartbeat scheduling
